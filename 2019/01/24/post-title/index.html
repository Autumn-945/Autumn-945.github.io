html5和css3的基础学习笔记

html5新增标签和属性以及事件

语义化标签

  1. html5语义化标签在谷歌火狐欧朋ie10等浏览器都是支持的，ie9仅支持部分，ie8及以下不支持。

  2. ie9虽然支持语义化标签，但是这些语义化标签都是行内元素， 解决：需要添加display:block;将行级元素转为块级元素即可。

  3. ie8完全不支持html5语义化标签，如何让ie8支持html5语义化标签？

    解决：

    第一种方式：手动创建标签 ：默认手动创建的标签的类型都是行级标签,也需要在样式中添加display:block;将行级元素转为块级元素。

    第二种方式：引入第三方插件 html5shiv.min.js

    这个文件兼容ie8及以下版本的html5标签

表单新增属性

type属性

  1. email提供了默认的电子邮箱的完整验证：要求必须包含@符号，同时必须包含服务器名称，如果不能满足验证，则会阻止当前验证的提交； ​

  2. tel它并不是来实现验证。它的本质目的是为了能够在移动端打开数字键盘。意味着数字键盘限制了用户只能输入数字。 如何查看效果：qq发送文件>>手机端使用qq来接收>>使用手机浏览器查看

  3. url验证只能输入合法的网址：必须包含http://

  4. number：只能输入数字(包含小数点)，不能输入其它的字符;max:最大值; min:最小值;value:默认值

  5. search：可以提供更人性化的输入体验 与text的不同点在当输入文本内容时，最后面会出现一个删除按钮，可以进行一次删除

  6. range 范围 可以用来 选择范围，有max、min、value 限制

  7. color 颜色 可以拾取颜色

  8. 日期与时间相关

    time 时间：时分秒

    date 日期 ：年月日

    datetime 日期时间 :大多浏览器不支持，只有safari支持。用于屏幕阅读器

    datetime-local 日期时间：大多浏览器都支持

    month 月份

    week 星期

表单新增其他属性

  1. placeholder：提示文本，提示占位

  2. autofocus 自动获取焦点

  3. autocomplete 提示功能 on:打开 off:关闭 默认是关闭

    实现提示功能有两个前提

    ​ 1.必须成功提交过，提交过才会有记录

    ​ 2.当前添加autocomplete属性的元素必须有name属性

  4. required 是要求必须输入，如果没有输入则不能成功提交

  5. pattern:正则表达式的正则验证​

  6. multiple 可以选择多个文件，在email中，multiple允许输入多个邮箱地址，以逗号分隔

  7. 在某种情况下，个别表单元素可能不能包含在form中：默认情况下表单中的数据也不能进行提交。

    form属性则可以使未包含在form表单的元素与指定form表单进行绑定。从而可以进行数据提交。

    用法：form:指定表单id,那么在将来指定id号的表单进行数据提交的时候，也会将当前表单元素的数据一起提交。

新增dom操作元素

获取dom元素

  • querySelector(传入选择器名称)

    querySelector:获取单个元素，如果获取的元素不止一个，那么只会返回满足条件的第一个元素

    参数要求：如果是类选择器，必须添加. 如果是id选择器， 必须添加# ,否则当成标签处理

  • querySelectorAll(传入选择器名称)

    querySelectorAll获取满足条件的所有元素–数组

操作元素类样式

  • classList:当前元素的所有样式列表-数组

    可以用item来获取里面的样式​

  • add:为元素添加指定名称的样式.一次只能添加一个样式

  • remove:为元素移除指定名称的样式(不是移除class属性 也就是说移除之后classs属性还是在的，只是类名被移除了)，一次也只能移除一个

  • toggle：切换元素的样式：如果元素之前没有指定名称的样式则添加。如果有则移除

  • contains:判断元素是否包含指定名称的样式，返回true/false

自定义属性

规范：

1.data-开头（所有自定义属性都是通过data-开头）

2.data-后必须至少有一个字符，多个单词使用-连接

建议：

1.名称应该都使用小写–不要包含任何的大写字符

2.名称中不要有任何的特殊符号

3.名称不要副作用纯数字

获取自定义属性值：通过dataset来获取

注意：将data-后面的单词使用camel（驼峰命名法）命名法连接，必须使用camel合法法获取值否则有可能无法获取到值,因为如果不使用驼峰命名法

  • onoffline:网络断开时触发

全屏接口

全屏操作的主要方法和属性

  1. requestFullScreen():开启全屏显示

    不同浏览器需要添加不同的前缀 chrome:webkit firefox:moz ie:ms opera:o

  2. cancelFullScreen():退出全屏显示:也添加前缀，在不同的浏览器下.退出全屏只能使用document来实现

  3. fullScreenElement:是否是全屏状态，也只能使用document进行判断

    这里注意不同浏览器大小写区别， 只有火狐遵循了驼峰命名法

FileReader：读取文件内容

读取方法：

1.readAsText():读取文本文件(可以使用txt打开的文件)，返回文本字符串，默认编码是UTF-8

2.readAsBinaryString():读取任意类型的文件。返回二进制字符串，这个方法不是用来读取文件 展示给用户看，

而是存储文件。例如：读取文件的内容，获取二进制数据，传递给后台，后台接收了数据之后，再将数据存储

3.readAsDataURL():读取文件获取一段以data开头的字符串，这段字符串的本质就是DataURL.

DataURL是一种将文件(一般指图像或能嵌入到文档的文件格式)嵌入到文档的方案

DataURL是将资源转换为base64编码的字符串形式，并且将这些内容直接存储在url中，

也就是说dataurl可以优化网站的加载速度和执行效率

例如：没有使用dataurl的img中的src里存的是图片文件在服务器中的路径，这个需要向服务器请求，

拖慢加载速度。使用dataurl之后，dataurl会将图片转换为base64编码字符串，并存储在url中，减少了

向服务器请求的过程，优化了网站的加载速度。

4.abort():中断读取

案例：即时预览

拖拽使用方法

拖拽哪个元素？

在h5中，为了使元素可拖动，就必须为该元素添加属性draggable = “true”.(图片和超链接默认就可以拖拽)

拖拽事件

/*应用于被拖拽元素的事件

​ ondrag 应用于拖拽元素，整个拖拽过程都会调用–持续

​ ondragstart 应用于拖拽元素，当拖拽开始时调用

​ ondragleave 应用于拖拽元素，当鼠标离开拖拽元素时调用

​ ondragend 应用于拖拽元素，当拖拽结束时调用*/

/*应用于目标元素的事件

​ *ondragenter 应用于目标元素，当拖拽元素进入时调用

​ ondragover 应用于目标元素，当停留在目标元素上时调用

​ ondrop 应用于目标元素，当在目标元素上松开鼠标时调用

（浏览器有默认阻止ondrop事件的行为，解决：必须在ondragover中阻止浏览器的默认行为： e.preventDefault(); ）

​ ondragleave 应用于目标元素，当鼠标离开目标元素时调用

任意元素拖拽到任意目标元素

实现对任意元素拖拽到任意目标元素，应当使用document来绑定拖拽事件

再通过事件源参数捕获来获取当前被拖拽的子元素

  • 通过全局变量来实现任意元素拖拽到任意目标元素

  • 不使用全局变量来实现任意元素拖拽到任意目标元素

    使用全局变量不是一个很好的注意，因为使用全局变量有几个缺点：
    ​ 1.全局变量任何值都对它赋值使用，不安全；
    ​ 2.全局变量生命周期比较长，会常驻内存，全局变量声明的太多时会造成内存泄露

    因此，事件源参数里有一个内置属性可以帮助我们来存储值以及获取值。

    这个属性为dataTransfer，用来实现数据的存储与获取；

    dataTransfer里面有两个方法，setDate()和getDate();

    setDate(format,data);

    getDate(format);

    format： 指数据类型 ：text/html text/uri-list

    data：指数据 : 一般来说是字符串值

    注意： 通过e.dataTransfer.setData存储的数据，只能在drop事件中获取

web存储

sessionStorage

sessionStorage的使用：存储数据到本地。存储的容量5mb左右。

  1. 这个数据本质是存储在当前页面的内存中-意味着其它页面和浏览器无法获取数据
  2. 它的生命周期为关闭当前页面，也就是说关闭页面，数据会自动清除

sessionStorage的方法:

  1. setItem(key,value):存储数据，以键值对的方式存储
  2. getItem(key):获取数据，通过指定名称的key获取对应的value值
  3. removeItem(key):删除数据，通过指定名称key删除对应的值
  4. clear():清空所有存储的内容

localStorage

localStorage的使用：

  • 存储的内容大概20mb
  • 不同浏览器不能共享数据。但是在同一个浏览器的不同窗口中可以共享数据
  • 永久生效，它的数据是存储在硬盘上，并不会随着页面或者浏览器的关闭而清除.如果想清除，必须手动清除

localStorage的方法：

  1. setItem(key,value):存储数据，以键值对的方式存储
  2. getItem(key):获取数据，通过指定名称的key获取对应的value值
  3. removeItem(key):删除数据，通过指定名称key删除对应的值
  4. clear():清空所有存储的内容、

应用缓存

应用缓存介绍

HTML5 引入了应用程序缓存，这意味着 web 应用可进行缓存，并可在没有因特网连接时进行访问。

应用程序缓存为应用带来三个优势：

  • 离线浏览 - 用户可在应用离线时使用它们
  • 速度 - 已缓存资源加载得更快
  • 减少服务器负载 - 浏览器将只从服务器下载更新过或更改过的资源。
  • chrome 可以通过chrome://appcache-internals/工具和离线（offline）模式来调试管理应用缓存

应用缓存的实现方法

  1. 如需启用应用程序缓存，必须在文档的 标签中包含 manifest 属性,值为一个名为appcache的文本文件，用来配置应用缓存的，需要手动创建这个文件， 如下创建的是名为demo的appcache文件。

    每个指定了 manifest 的页面在用户对其访问时都会被缓存。如果未指定 manifest 属性，则页面不会被缓存（除非在 manifest 文件中直接指定了该页面）。

    manifest 文件的建议的文件扩展名是：”.appcache”。

    请注意，manifest 文件需要配置正确的 MIME-type，即 “text/cache-manifest”。必须在 web 服务器上进行配置。配置方法自行百度

  2. manifest文件

    manifest 文件是简单的文本文件，它告知浏览器被缓存的内容（以及不缓存的内容）。

    manifest 文件可分为三个部分：

    • CACHE MANIFEST - 在此标题下列出的文件将在首次下载后进行缓存

    • NETWORK - 在此标题下列出的文件需要与服务器的连接，且不会被缓存

    • FALLBACK - 在此标题下列出的文件规定当页面无法访问时的回退页面（比如 404 页面）

更新缓存

一旦应用被缓存，它就会保持缓存直到发生下列情况：

  • 用户清空浏览器缓存
  • manifest 文件被修改（参阅下面的提示）
  • 由程序来更新应用缓存

重要的提示：应用的缓存会在其 manifest 文件更改时被更新。如果您编辑了一幅图片，或者修改了一个 JavaScript 函数，这些改变都不会被重新缓存。

说明：更新注释行中的日期和版本号是一种使浏览器重新缓存文件的办法。

css3新增选择器

属性选择器

a) E[attribute] 表示存在attr属性即可；

div[class]

b) E[attr=val] 表示属性值完全等于val；

div[class=mydemo]

c) E[attr*=val] 表示的属性值里包含val字符并且在“任意”位置；

div[class*=mydemo]

d) E[attr^=val] 表示的属性值里包含val字符并且在“开始”位置；

div[class^=mydemo]

e) E[attr$=val] 表示的属性值里包含val字符并且在“结束”位置；

div[class$=demos]

伪类选择器

a) 之前学习的：a:hover a:link a:active a:visited

b) 以某元素相对于其父元素或兄弟元素的位置来获取无素的结构伪类

a) E:first-child:查找E这个元素的父元素的第一个子元素E

b) E:last-child:最后一个子元素

c) E:nth-child(n): 第n个子元素，计算方法是E元素的全部兄弟元素

d) E:nth-last-child(n): 同E:nth-child(n) 相似，只是倒着计算

e) E:nth-child(even): 所有的偶数

f) E:nth-child(odd): 所有的奇数

g) E:nth-of-type(n):指定类型

h) E:empty 选中没有任何子节点的E元素，注意，空格也算子元素

i) E:target 结合锚点进行使用，处于当前锚点的元素会被选中

j) 重点说明：n遵循线性变化，其取值0、1、2、3、4、… 但是当n<=0时，选取无效

伪元素选择器

a) 重点：E::before、E::after

i. 是一个行内元素，需要转换成块:display:block float:** position:

ii. 必须添加content,哪怕不设置内容，也需要content:””

iii. E:after、E:before 在旧版本里是伪类，在新版本里是伪元素，新版本下E:after、E:before会被自动识别为E::after、E::before，按伪元素来对待，这样做的目的是用来做兼容处理

iv. E::before: 定义在一个元素的内容之前插入content属性定义的内容与样式

v. E::after: 定义在一个元素的内容之后插入content属性定义的内容与样式

vi. 注意：

  1. IE6、IE7与IE8（怪异模式Quirks mode）不支持此伪元素
  2. CSS2中 E:before或者E:after，是属于伪类的，并且没有伪元素的概念，CSS3中 提出伪元素的概念 E::before和E::after，并且归属到了伪元素当中，伪类里就不再存在E:before或者 E:after伪类

b) E::first-letter文本的第一个字母或字(不是词组)

c) E::first-line 文本第一行

d) E::selection 可改变选中文本的样式

css3渐变

线性渐变

  • linear-gradient线性渐变指沿着某条直线朝一个方向产生渐变效果

    a) 语法：

    linear-gradient( [ || ,]? , [, ]* )

    b) 参数说明：

    Ø 第一个参数表示线性渐变的方向，

  1. to left：设置渐变为从右到左。相当于: 270deg;

  2. to right：设置渐变从左到右。相当于: 90deg;

  3. to top：设置渐变从下到上。相当于: 0deg;

  4. to bottom：设置渐变从上到下。相当于: 180deg。这是默认值，等同于留空不写。也可以直接指定度数，如45deg

    Ø 第二个参数是起点颜色,可以指定颜色的位置

    Ø 第三个参数是终点颜色，你还可以在后面添加更多的参数，表示多种颜色的渐变
